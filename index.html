Overview

Behave for JIRA is a powerful and easy to use agile testing and requirements discovery tool. It allows users to take TDD to the next step, Acceptance Test Driven Development. Specific needs of a software product are expressed as acceptance criteria on User Stories, invaluable for discovering the real requirements through discussion. Further more the acceptance criteria can be automated as tests using Cucumber to provide instant feedback on software development progress.

Benefits

  • Avoids unnecessary rework due to issues being found at the requirements stage and not during testing
  • Testers know what has been tested and can focus their testing in other areas
  • No additional skilled project remembers required, Acceptance Criteria are written in natural language so anyone can write them.

What's the scenario?

Acceptance Criteria are written as Scenarios which define clear contexts and events in the application and there expected results. Written in a clear and logical "Given, When, Then" natural language syntax called Gherkin. Natural language menas simple to understand for all stakeholders. Step by step definition is concise enough to be implemented by developers.

Test Automation

Don't let your Test engineers become expensive keyboard automation tools, Behave for JIRA can generate acceptance tests for the leading open source BDD automation framework, Cucumber. Cucumber supports a number of programming languages including: Java, Ruby, and Scala.

To automate or not to automate?

Test automation is not suited to every situation - Behave for JIRA has it covered. A mixture of automated and manual testing can be achieved by marking individual acceptance criteria as "Manual" to stop them being run by Cucumber.
