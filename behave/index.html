Overview

Behave for JIRA is a powerful and easy to use agile testing and requirements discovery tool. It allows users to take TDD to the next step, Acceptance Test Driven Development.

Specific needs of a software product are expressed as acceptance criteria on User Stories, invaluable for discovering the real requirements through discussion. The acceptance criteria can then be automated as tests with Cucumber to provide instant feedback on software development progress.

Feature Highlights

  • Add scenarios to individual User stories (or any JIRA Issue) as acceptance tests
  • Scenarios are written in a natural language called Gherkin (Given When Then) so any one can write them
  • Simple test management with the choice between manual and automated testing
  • Scenarios/Acceptance tests can be automated using Cucumber, a popular open source testing tool
  • No vendor lock in; you can export your tests from JIRA and continue to use them with just Cucumber
  • Selectively run automated tests based on if the parent JIRA issue is closed or in progress.

Benefits

As a product owner

  • Be clear and upfront about what you want your software to achieve
  • Use simple natural language to include all stakeholders in the discussion
  • Know when your software meets the acceptance criteria
  • Streamline your interaction with developers - they know exactly which scenarios they need to cover

As a developer

  • Understand what your clients expect in defined and testable terms before writing any code
  • Focus on implementing the scenario at hand, rather than endless back and forth clarifying requirements
  • Know when you've written the code the client wants - and the client knows it too!

As a tester

  • Get the context right from the start about how to test the software - test in line with stakeholders expectations
  • Full regression protection at a scenario level - know when end-to-end functionality gets broken

As a software user

  • See living documentation that reflects the needs of the user rather than the eccentricities of the development team!
  • Be a part of the discussion for future features and fixes in the software

Simplicity

Acceptance Criteria are written as Scenarios which define clear contexts and events in the application and there expected results. Written in a clear and logical "Given, When, Then" natural language syntax called Gherkin. Natural language menas simple to understand for all stakeholders. Step by step definition is concise enough to be implemented by developers.

Writing acceptance tests as scenarios in simple natural language allows everyone in a team to read and understand the requirements. If there is any miss understanding over a scenario, a discussion can take place between all the team members and the stakeholders. The scenario can be rewritten to include any clarifications discovered during the discussion.

Automation

Testing Army

Don't let your test engineers become expensive keyboard automation tools, Behave for JIRA can generate acceptance tests for the leading open source BDD (Behavior driven development) automation framework, Cucumber. Cucumber supports a number of programming languages including: Java, Ruby, and Scala.

To automate or not to automate?

Automated testing is not suited to every situation - Behave for JIRA has it covered. A mixture of automated and manual testing can be achieved by marking individual acceptance criteria as "Manual" to stop them being run as Automated tests.

Integration

The Atlassian Cucumber integration doesn't stop at JIRA, Bamboo users can take advantage of the Cucumber Report Plugin. The plugin is available for download on the Atlassian Marketplace
